# ミニバンキングシステム MVP定義

このドキュメントは、学習用ミニバンキングシステムで最初に実装する範囲と、意図的に実装しない範囲を定義します。金融システムは対象領域が広いため、まずはお金の移動・整合性・監査性を小さく確認できる範囲に絞ります。

## MVPの目的

MVPでは、次の流れを安全に扱えることを目指します。

1. ログインユーザーを作成する
2. 顧客を登録する
3. 顧客に口座を作成する
4. 口座へ入金する
5. 口座から出金する
6. 口座間で振込する
7. 残高と取引履歴を確認する
8. 重要操作の監査ログを確認する

## MVPで実装する機能

| 機能 | 概要 | 主な成功条件 |
| --- | --- | --- |
| ユーザー登録 | システムへログインできるユーザーを作成する | ログインIDが重複しない。パスワードはハッシュ化して保存する。 |
| 認証 | ユーザーがログインして操作できるようにする | 未認証ユーザーは口座・取引を操作できない。 |
| 顧客登録 | 銀行サービスを利用する顧客情報を登録する | 顧客IDが発行され、状態が有効になる。 |
| 口座作成 | 顧客に普通預金口座を作成する | 口座番号が一意で、初期残高は0円になる。 |
| 入金 | 口座残高を増やす | 金額が正の整数で、取引履歴と監査ログが残る。 |
| 出金 | 口座残高を減らす | 残高不足の場合は拒否し、成功時は残高がマイナスにならない。 |
| 振込 | 口座間で資金を移動する | 振込元の出金と振込先の入金が同時に成功または失敗する。 |
| 残高照会 | 口座の現在残高を確認する | 認可されたユーザーだけが対象口座を照会できる。 |
| 取引履歴照会 | 口座の取引履歴を確認する | 取引が時系列で確認でき、残高変更後金額も分かる。 |
| 監査ログ記録 | 重要操作の操作記録を残す | 誰が、いつ、何を、どの結果で実行したか追跡できる。 |

## MVPで扱う主な異常系

- 未認証ユーザーによる操作は拒否する。
- 権限のない口座照会・入出金・振込は拒否する。
- 0円以下の入金・出金・振込は拒否する。
- 残高不足の出金・振込は拒否する。
- 停止中または解約済み口座の入出金・振込は拒否する。
- 存在しない口座への操作は拒否する。
- 同じ振込依頼が再送された場合、二重送金しない。
- 処理途中でエラーが発生した場合、片方の口座だけ残高が変わらないようにする。

## MVPでは作らない機能

| 対象外機能 | 理由 |
| --- | --- |
| 外部銀行接続 | 学習初期では外部連携より内部整合性を優先するため。 |
| クレジットカード決済 | 与信、請求、加盟店精算など別ドメインが大きいため。 |
| 本人確認書類アップロード | ファイル管理、審査ワークフロー、個人情報管理が重くなるため。 |
| 複雑な金利計算 | まずは入出金・振込の基本取引を優先するため。 |
| 投資商品 | 株式、投資信託、FXは銀行口座とは異なる業務知識が必要なため。 |
| 高度な不正検知AI | 監査ログと基本的な制約を先に整えるため。 |
| 多通貨対応 | 通貨換算や為替レート管理が必要になるため、MVPではJPYのみとする。 |
| 共同名義口座 | 権限・同意・代表者などの設計が複雑になるため。 |

## 完了条件

MVPは、以下を満たしたら完了とします。

- 入金、出金、振込で残高が正しく更新される。
- すべての残高変更に取引履歴が残る。
- すべての重要操作に監査ログが残る。
- 残高不足や不正な金額などの異常系が拒否される。
- 振込処理が途中で片方だけ成功しない。
- 同一の振込依頼を再送しても二重送金されない。
- 主要な正常系・異常系の自動テストがある。
