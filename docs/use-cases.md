# ミニバンキングシステム ユースケース

このドキュメントは、MVPで想定する利用シナリオを正常系と異常系に分けて整理します。実装前に失敗時の扱いを明確にすることで、金融システムに必要な整合性と監査性を守りやすくします。

## 登場人物

- 顧客: 自分の口座を利用するユーザー。
- 管理者: 顧客登録、口座作成、監査ログ確認などを行うユーザー。
- システム: 残高更新、取引履歴作成、監査ログ記録を行うアプリケーション。

## UC-001 顧客を登録する

### 正常系

1. 管理者が認証済み状態で顧客登録を開始する。
2. 管理者が氏名、メールアドレス、電話番号、住所を入力する。
3. システムが入力値を検証する。
4. システムが顧客を有効状態で登録する。
5. システムが監査ログに登録成功を記録する。

### 異常系

- 必須項目が不足している場合、登録を拒否する。
- 権限のないユーザーが実行した場合、登録を拒否する。
- 登録に失敗した場合、監査ログに失敗を記録する。

## UC-002 口座を作成する

### 正常系

1. 管理者が対象顧客を選択する。
2. システムが顧客の状態を確認する。
3. システムが一意な口座番号を発行する。
4. システムが初期残高0円の普通預金口座を作成する。
5. システムが監査ログに口座作成成功を記録する。

### 異常系

- 対象顧客が存在しない場合、口座作成を拒否する。
- 対象顧客が停止中または解約済みの場合、口座作成を拒否する。
- 口座番号の発行に失敗した場合、口座を作成しない。

## UC-003 入金する

### 正常系

1. 認証済みユーザーが入金対象口座と金額を指定する。
2. システムが口座の存在、状態、操作権限を確認する。
3. システムが金額が正の整数であることを確認する。
4. システムが口座残高を増やす。
5. システムが入金取引を作成する。
6. システムが監査ログに入金成功を記録する。

### 異常系

- 対象口座が存在しない場合、入金を拒否する。
- 対象口座が停止中または解約済みの場合、入金を拒否する。
- 金額が0円以下の場合、入金を拒否する。
- 権限のないユーザーが実行した場合、入金を拒否する。

## UC-004 出金する

### 正常系

1. 顧客が自分の口座と出金金額を指定する。
2. システムが口座の存在、状態、操作権限を確認する。
3. システムが金額が正の整数であることを確認する。
4. システムが残高不足でないことを確認する。
5. システムが口座残高を減らす。
6. システムが出金取引を作成する。
7. システムが監査ログに出金成功を記録する。

### 異常系

- 残高不足の場合、出金を拒否し、残高を変更しない。
- 金額が0円以下の場合、出金を拒否する。
- 対象口座が停止中または解約済みの場合、出金を拒否する。
- 権限のないユーザーが実行した場合、出金を拒否する。

## UC-005 口座間で振込する

### 正常系

1. 顧客が振込元口座、振込先口座、金額、冪等性キーを指定する。
2. システムが振込元口座と振込先口座の存在・状態を確認する。
3. システムが振込元口座の操作権限を確認する。
4. システムが金額が正の整数であることを確認する。
5. システムが残高不足でないことを確認する。
6. システムが振込依頼を受付済みとして作成する。
7. システムが同一トランザクション内で振込元残高を減らし、振込先残高を増やす。
8. システムが振込出金取引と振込入金取引を作成する。
9. システムが振込依頼を成功に更新する。
10. システムが監査ログに振込成功を記録する。

### 異常系

- 残高不足の場合、振込を拒否し、どちらの残高も変更しない。
- 同じ冪等性キーの成功済み依頼がある場合、既存の結果を返し、再実行しない。
- 振込先口座が存在しない場合、振込を拒否する。
- 振込元または振込先口座が停止中・解約済みの場合、振込を拒否する。
- 処理途中でエラーが発生した場合、トランザクションをロールバックする。

## UC-006 残高を照会する

### 正常系

1. 顧客が自分の口座を選択する。
2. システムが認証と認可を確認する。
3. システムが現在残高を返す。
4. 必要に応じて監査ログに照会を記録する。

### 異常系

- 他人の口座を照会しようとした場合、照会を拒否する。
- 口座が存在しない場合、照会を拒否する。

## UC-007 取引履歴を照会する

### 正常系

1. 顧客が口座と検索条件を指定する。
2. システムが認証と認可を確認する。
3. システムが取引履歴を時系列で返す。

### 異常系

- 権限のない口座の履歴照会は拒否する。
- 検索期間が不正な場合、照会を拒否する。

## UC-008 監査ログを確認する

### 正常系

1. 管理者が監査ログの検索条件を指定する。
2. システムが管理者権限を確認する。
3. システムが条件に一致する監査ログを返す。

### 異常系

- 管理者権限がない場合、監査ログ照会を拒否する。
- 検索条件が広すぎる場合、期間指定を求める。
